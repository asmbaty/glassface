# GlassFace

A tiny native macOS app that shows your **front camera feed as a semi-transparent,
full-screen overlay** — while you keep using your Mac completely normally. The overlay
floats above everything, on every Space, and **passes all clicks and keystrokes through**
to the apps underneath.

## Requirements
- macOS 11+
- Xcode Command Line Tools (`xcode-select --install`) for `swiftc`

## Build
```bash
./build.sh
```
This compiles the sources in `Sources/` (plus the `main.swift` entry point) and assembles
`GlassFace.app` (no Xcode project needed).

## Run
```bash
open GlassFace.app
```
On first launch macOS asks for **camera permission** — approve it. The transparent feed
then covers your whole screen.

## Menu bar & hotkeys
GlassFace lives in the **macOS menu bar** (camera icon, top-right). Click it for a menu
showing the current opacity, with Increase / Decrease / Quit. Global hotkeys work from any app:

| Action | Hotkey |
|--------|--------|
| Increase opacity | **⌃⌥⌘ =** |
| Decrease opacity | **⌃⌥⌘ -** |
| Set opacity 10%–90% | **⌃⌥⌘ 1** … **⌃⌥⌘ 9** |
| Set opacity 100% | **⌃⌥⌘ 0** |
| Quit | **⌃⌥⌘ Q** |

(Or `killall GlassFace` from a terminal.) Opacity changes by 10% per press, clamped to 0–100%.

## Heart gesture ❤️
Make a **two-hand heart** at the camera (index fingertips touching at the top, thumb tips
touching at the bottom) and GlassFace draws a pulsing red heart over the feed, positioned
between your hands. Drop the gesture and it fades away. Toggle it on/off from the menu bar
(**Heart Gesture**). Detection uses Apple's **Vision** hand-pose framework on-device — no extra
permission and no third-party dependencies. Thresholds live in `detectHeart()` in `main.swift`
if you want to make it stricter or looser.

## Customize
Almost everything tunable lives in one place: **`Sources/AppConfiguration.swift`**.
- **Starting transparency / step:** `defaultOpacity` (`0.0` = invisible, `1.0` = opaque; default
  `0.35`) and `opacityStep` (hotkey increment).
- **Heart:** `heartSize`, the gesture thresholds (`maxIndexTipGap`, `maxThumbTipGap`,
  `minJointConfidence`), and `frameProcessingStride`.
- **Camera:** defaults to the built-in front (FaceTime) camera; falls back to any available
  camera. The feed is **mirrored** (selfie view). See `CameraService.frontCamera()`.
- **Multiple displays:** an overlay is created for every connected screen automatically.
- **Icon:** generated by `make_icon.swift` into `AppIcon.icns` (delete the `.icns` to regenerate
  on next build).

## How it works
The code is split into small single-responsibility types under `Sources/`, wired together by
`AppCoordinator` (the composition root); `main.swift` is just the entry point.

| Concern | Type(s) |
|---------|---------|
| Configuration (all tunables) | `AppConfiguration` |
| Camera capture & frame delivery | `CameraService` (`FrameConsumer`) |
| Heart detection (Vision hand pose) | `HeartGestureRecognizer`, `HeartGesture` |
| On-screen overlay | `OverlayCoordinator`, `ScreenOverlay`, `OverlayWindow`, `HeartOverlayLayer`, `HeartShape` |
| Global hot keys (Carbon) | `HotKeyCenter` |
| Menu bar | `StatusMenuController`, `GhostImageRenderer` |
| Errors | `AlertPresenter` |
| Wiring / orchestration | `AppCoordinator` |

Under the hood: **AppKit** borderless `NSWindow` per screen (`level = .screenSaver`,
`ignoresMouseEvents = true` — the key to staying usable); **AVFoundation**
`AVCaptureSession` → `AVCaptureVideoPreviewLayer` with reduced `opacity`; **Carbon**
`RegisterEventHotKey` for global shortcuts (no Accessibility permission); **Vision**
`VNDetectHumanHandPoseRequest` on the raw `AVCaptureVideoDataOutput` frames to spot the
two-hand heart.

## Notes
- The overlay sits above the menu bar and most apps. It does **not** appear above another
  app's native full-screen (green-button) mode on its own Space; use it on the normal desktop.
- If the camera permission prompt keeps reappearing, ensure `codesign` succeeded during build.
